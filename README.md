# syntactic_analysis
ИТиКИПиИЗ лаб. 2

Разработать формальную грамматику для предметно-ориентированного языка (DSL) запросов к базе данных и реализовать синтаксический анализатор, строящий деревья разбора для корректных выражений.

input.txt - тестовый файл с запросами
ast_nodes.py - Определяет структуру узла абстрактного синтаксического дерева (Node). Узел имеет имя, значение и дочерние узлы.
tokens.py - Содержит перечисление типов токенов (TokenType) и класс токена (Token). Токен хранит тип, нормализованное значение, позицию и исходное слово.
lexer.py - Лексический анализатор. Разбивает входную строку на токены.
parser.py - Синтаксический анализатор. Принимает список токенов от лексера и строит AST согласно грамматике запроса:
команда -> объект (книги, романы) -> опциональные уточнения (авторы, "все", "первые N") -> фильтры (по годам, теме, диапазонам)
кс - грамматика реализована рекурсивным спуском.
errors.py - Собственный класс ошибки ParseError, расширяющий Exception. При печати показывает проблемный токен и его позицию.
utils.py - Вспомогательная функция print_result() для красивого вывода результата разбора: дерево AST при успехе, сообщение об ошибке при неудаче.
